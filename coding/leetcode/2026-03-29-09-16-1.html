LeetCode #1 - 两数之和

Easy 哈希表 数组 | 今日精讲 · 第160期

📋 题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且 同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。

示例 1

输入：nums = [2,7,11,15], target = 9

输出：[0,1]

解释：因为 nums[0] + nums[1] == 9，返回 [0, 1]。

示例 2

输入：nums = [3,2,4], target = 6

输出：[1,2]

示例 3

输入：nums = [3,3], target = 6

输出：[0,1]

💡 核心思路

对于当前元素 nums[i]，我们需要找到另一个数 target - nums[i]。哈希表让查找从 O(n) 降到 O(1)。一边遍历一边存入哈希表，如果表中已有所需的「补数」，直接返回结果。

🎬 算法执行过程（点击步骤或代码行联动查看）

步骤 1: 初始化哈希表

创建一个空字典，用于存储「数值 → 下标」的映射关系

hashmap = {}

步骤 2: 开始遍历数组

使用 enumerate 同时获取索引 i 和数值 num

i=0, num=2, nums=[2,7,11,15]

步骤 3: 计算补数

计算需要找的另一个数：complement = target - num = 9 - 2 = 7

complement = 7

步骤 4: 检查补数是否在哈希表中

哈希表为空，7 不在表中，继续执行

7 not in {} → True

步骤 5: 存入当前数值和下标

将当前数值 2 和下标 0 存入哈希表，供后续元素查找

hashmap = {2: 0}

步骤 6: 处理下一个元素

进入下一轮循环，i=1, num=7

i=1, num=7

步骤 7: 再次计算补数

complement = 9 - 7 = 2

complement = 2

步骤 8: 找到匹配！

补数 2 在哈希表中！返回当前下标 1 和哈希表中的下标 0

2 in {2: 0} → return [hashmap[2], 1] = [0, 1]

Python 实现

1class Solution:
2    def twoSum(self, nums: List[int], target: int) -> List[int]:
3        # 哈希表：存储 数值 -> 下标 的映射
4        hashmap = {}
5        for i, num in enumerate(nums):
6            complement = target - num
7            if complement in hashmap:
8                return [hashmap[complement], i]
9            hashmap[num] = i
10        return []  # 题目保证有解，这行不会执行

📊 复杂度分析

方案 时间复杂度 空间复杂度 说明
暴力枚举 O(n²) O(1) 两重循环遍历所有组合
哈希表（推荐） O(n) O(n) 一次遍历，空间换时间

最优选择：哈希表法虽然使用了 O(n) 的额外空间，但将时间从 O(n²) 优化到 O(n)，在绝大多数场景下是更好的选择。

🎯 关键点总结

  • 边查边存：遍历的同时将已处理的元素存入哈希表，这样可以在一次遍历中完成查找
  • 避免重复使用元素：先查哈希表再存当前元素，确保不会用同一个元素两次
  • 空间换时间：这是典型的用额外空间换取时间优化的案例
  • enumerate 技巧：Python 中同时获取索引和值的最佳实践

📝 面试高频追问

Q1: 如果数组是有序的，你会怎么做？

A: 使用双指针法。一个指针在头部，一个指针在尾部，根据两数之和与 target 的大小关系移动指针。时间 O(n)，空间 O(1)。

Q2: 如果有多个答案，要求输出所有组合？

A: 不能用 return 直接返回，需要收集所有结果。可以用 defaultdict 存储相同值的所有下标，找到匹配后遍历组合。

Q3: 数据量很大（10^9 级别）怎么处理？

A: 考虑分布式处理，或使用外部排序后双指针。哈希表受限于单机内存，需要考虑分片存储。
