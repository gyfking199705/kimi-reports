🚩 LeetCode 75 - 颜色分类

Medium 数组 双指针 荷兰国旗 原地排序

📋 题目描述

给定一个包含红色、白色和蓝色、共 n 个元素的数组 nums ，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。

我们使用整数 0、1 和 2 分别表示红色、白色和蓝色。

必须在不使用库内置的 sort 函数的情况下解决这个问题。

🎯 核心考点

荷兰国旗问题的经典实现。使用三指针法，在 O(n) 时间和 O(1) 空间内完成原地排序。关键是维护三个区间：[0,p0)全是0，[p0,curr)全是1，(p2,n-1]全是2。

🎬 算法执行过程（点击步骤或代码行查看对应关系）

步骤 0: 初始化三指针
p0 = 0 (0的右边界), curr = 0 (当前遍历位置), p2 = n-1 (2的左边界)
初始数组: [2,0,2,1,1,0]
步骤 1: 遍历数组
当 curr ≤ p2 时继续遍历
curr=0, p2=5, 继续遍历
步骤 2: 遇到 2 (nums[curr]==2)
交换 nums[curr] 和 nums[p2]，p2 左移。注意：curr 不变，需要重新检查交换后的值
交换 nums[0]↔nums[5]: [0,0,2,1,1,2], p2=4
步骤 3: 遇到 0 (nums[curr]==0)
交换 nums[curr] 和 nums[p0]，p0 和 curr 都右移
交换 nums[0]↔nums[0]: [0,0,2,1,1,2], p0=1, curr=1
步骤 4: 再次遇到 0
交换后 p0=2, curr=2
p0=2, curr=2
步骤 5: 遇到 2
交换 nums[2]↔nums[4]: [0,0,1,1,2,2], p2=3
注意 curr 保持为 2，需要重新检查
步骤 6: 遇到 1
curr 直接右移，不做交换
curr=3, nums[3]=1
步骤 7: 再次遇到 1
curr 右移到 4，此时 curr > p2，遍历结束
最终结果: [0,0,1,1,2,2] ✓

🔑 为什么遇到 2 时 curr 不移动？

当 nums[curr] == 2 时，我们将它与 nums[p2] 交换。但交换过来的元素可能是 0 或 1，需要在本轮继续处理。而遇到 0 时，交换过来的一定是 1（因为 p0 ≤ curr），所以可以直接移动 curr。
