LeetCode #128 - 最长连续序列

📋 题目描述

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 1：
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
示例 2：
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9

约束条件：

  • 0 ≤ nums.length ≤ 10⁵
  • -10⁹ ≤ nums[i] ≤ 10⁹

🧠 核心思路

这道题的关键在于如何实现 O(n) 的时间复杂度。如果直接排序，时间复杂度是 O(n log n)，不符合要求。

核心洞察：

  • 使用哈希集合（HashSet）存储所有数字，实现 O(1) 查找
  • 只从序列的起点开始扩展：如何判断一个数是否是起点？检查 num-1 是否在集合中
  • 如果是起点，就一直向后查找 num+1, num+2... 直到不连续为止

这样做的原因是：每个连续序列只会被遍历一次，因为每个数只会作为起点被检查一次，或者被跳过（因为不是起点）。

📊 复杂度分析

复杂度类型 值 说明
时间复杂度 O(n) 每个元素最多被访问两次：一次构建集合，一次作为序列的一部分
空间复杂度 O(n) 需要哈希集合存储所有元素

💡 关键点总结

  • 优化的核心：只从序列起点开始扩展，避免重复遍历
  • 如何判断起点：num-1 不在集合中
  • 为什么 O(n)：虽然有两层循环，但内层循环每个元素只会进入一次
  • 常见错误：忘记检查 num-1 是否存在，导致对每个数都进行序列扩展，退化为 O(n²)
