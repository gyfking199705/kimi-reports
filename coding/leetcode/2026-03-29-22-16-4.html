LeetCode #4 - 寻找两个正序数组的中位数Hard

二分查找 · 数组 · 分治算法

📋 题目描述

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log(m+n)) 。

示例 1：

  • 输入：nums1 = [1,3], nums2 = [2]
  • 输出：2.00000
  • 解释：合并数组 = [1,2,3] ，中位数 2

示例 2：

  • 输入：nums1 = [1,2], nums2 = [3,4]
  • 输出：2.50000
  • 解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5

💡 核心思路

🔍
二分查找的巧妙应用
问题本质是找第k小的数。通过对较短的数组二分，将问题规模对半缩小，达到 O(log(min(m,n))) 复杂度。
✂️
排除法的精髓
每次比较两个数组的第 k/2 个元素，较小的一半不可能包含第k小元素，直接排除。
⚡
边界情况处理
处理空数组、数组耗尽、k=1等边界，确保代码鲁棒性。

执行步骤（示例：nums1=[1,3,5,7], nums2=[2,4,6,8]）

1
初始化与数组交换
确保nums1是较短的数组，优化二分范围。若nums1更长，则交换两个数组。
nums1=[1,3,5,7](m=4), nums2=[2,4,6,8](n=4)
2
计算总长度与中点
总长度8，half=(8+1)//2=4，表示左半部分应有4个元素。
total=8, half=4
3
二分查找初始化
在nums1上进行二分，left=0, right=4（nums1的长度）。
left=0, right=4
4
计算分割点i
i=(0+4)//2=2，表示从nums1取前2个元素到左半部分。
i=2, nums1分割: [1,3] | [5,7]
5
计算分割点j
j=half-i=4-2=2，表示从nums2取前2个元素到左半部分。
j=2, nums2分割: [2,4] | [6,8]
6
获取边界值
计算四个关键值：nums1_left=3, nums1_right=5, nums2_left=4, nums2_right=6
left_max=max(3,4)=4, right_min=min(5,6)=5
7
验证分割正确性
检查 nums1_left(3) ≤ nums2_right(6) 且 nums2_left(4) ≤ nums1_right(5)。条件满足！
3≤6 ✓, 4≤5 ✓
8
判断长度奇偶性
总长度8为偶数，中位数=(左半最大值+右半最小值)/2
total%2=0 → 偶数处理
9
计算左半最大值
左半部分的最大值是 max(nums1_left, nums2_left) = max(3,4) = 4
max_left = 4
10
计算右半最小值
右半部分的最小值是 min(nums1_right, nums2_right) = min(5,6) = 5
min_right = 5
11
返回中位数
最终结果 (4+5)/2 = 4.5
median = 4.5

Python 实现



 1class Solution:


 2    def findMedianSortedArrays(self, nums1: list[int], nums2: list[int]) -> float:


 3        # 确保 nums1 是较短的数组，优化二分范围


 4        if len(nums1) > len(nums2):


 5            nums1, nums2 = nums2, nums1


 6        


 7        m, n = len(nums1), len(nums2)


 8        total = m + n


 9        half = (total + 1) // 2  # 中位数分割点


10        


11        # 在 nums1 上进行二分查找


12        left, right = 0, m


13        


14        while left <= right:


15            i = (left + right) // 2  # nums1 的分割点


16            j = half - i           # nums2 的分割点


17            


18            # 获取分割点左右的值（处理边界）


19            nums1_left = float('-inf') if i == 0 else nums1[i - 1]


20            nums1_right = float('inf') if i == m else nums1[i]


21            nums2_left = float('-inf') if j == 0 else nums2[j - 1]


22            nums2_right = float('inf') if j == n else nums2[j]


23            


24            # 检查是否找到正确的分割点


25            if nums1_left <= nums2_right and nums2_left <= nums1_right:


26                # 找到正确分割点，计算中位数


27                if total % 2 == 0:


28                    max_left = max(nums1_left, nums2_left)


29                    min_right = min(nums1_right, nums2_right)


30                    return (max_left + min_right) / 2.0


31                else:


32                    return max(nums1_left, nums2_left)


33            


34            elif nums1_left > nums2_right:


35                # nums1 的左半部分太大，向左移动


36                right = i - 1


37            else:


38                # nums1 的左半部分太小，向右移动


39                left = i + 1


40        


41        return 0.0  # 理论上不会执行到这里

📊 复杂度分析

维度复杂度说明
时间复杂度O(log(min(m,n)))在较短的数组上进行二分查找，每次排除一半元素
空间复杂度O(1)只使用了常数额外空间

🎯 关键点总结

1. 转换问题：将找中位数转换为在虚拟合并数组中找分割点，使左右两部分元素数量平衡。

2. 二分策略：在较短的数组上二分，通过分割点 i 推导出 nums2 的分割点 j=half-i。

3. 判定条件：当 nums1_left ≤ nums2_right 且 nums2_left ≤ nums1_right 时，找到正确分割点。

4. 边界处理：使用正负无穷处理数组分割边界，简洁优雅。
