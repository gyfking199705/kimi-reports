LeetCode #141 - 环形链表

Linked List Cycle · Easy · 快慢指针经典入门

📋 题目描述

给你一个链表的头节点 head，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：pos 不作为参数进行传递。仅仅是为了标识链表的实际情况。

如果链表中存在环，则返回 true；否则，返回 false。

7class Solution:


8    def hasCycle(self, head: Optional[ListNode]) -> bool:


9        # 初始化快慢指针


10        slow = fast = head


11        


12        while fast and fast.next:


13            # 慢指针走一步


14            slow = slow.next


15            # 快指针走两步


16            fast = fast.next.next


17            


18            # 相遇则说明有环


19            if slow == fast:


20                return True


21        


22        return False

📊 复杂度分析

维度 值 说明
时间复杂度 O(n) 最坏情况下遍历整个链表一次。当快指针追上慢指针时，最多需要 n 步。
空间复杂度 O(1) 只使用两个指针，无需额外数据结构。这是快慢指针相比哈希表的优势！
为什么快指针走2步？ 最优 走2步是时间和实现复杂度的平衡。走更多步可能跳过，走1步无法区分环。

💡 核心原理解析

为什么快慢指针一定会相遇？

假设链表有环，设环的长度为 L。当慢指针进入环时，快指针已经在环中某个位置。

此时快指针追赶慢指针，每次循环：

  • 慢指针前进1步，快指针前进2步
  • 两者距离 减少1（相对速度为1）

由于相对速度为1，快指针一定会追上慢指针，且最多需要 L 步。


无环的情况

如果链表无环，快指针会先到达链表末尾（fast 或 fast.next 为 None），循环结束返回 False。
