反转链表 | Reverse Linked List

📋 题目描述

给你单链表的头节点 head，请你反转链表，并返回反转后的链表的头节点。

核心考点：链表指针操作的经典入门题，掌握迭代和递归两种解法，理解指针的「前驱-当前-后继」关系。

示例

输入：head = [1, 2, 3, 4, 5]

输出：[5, 4, 3, 2, 1]

输入：head = [1, 2]

输出：[2, 1]

输入：head = []

输出：[]

约束条件

  • 链表中节点的数目范围是 [0, 5000]
  • -5000 <= Node.val <= 5000

💡 核心思路解析

迭代法三指针技巧

反转链表的核心是改变每个节点的 next 指向。我们需要三个指针协同工作：

  • prev - 已反转部分的头节点（初始为 None）
  • curr - 当前正在处理的节点
  • next_temp - 临时保存 curr 的后继，防止断链后丢失
关键操作顺序：保存后继 → 反转指向 → 前移指针。
顺序不能错！如果先反转，会丢失后继节点的引用。

递归法思路

递归到底层（最后一个节点），然后逐层返回时反转指针：

  1. 递归到链表末尾，找到新的头节点
  2. 让当前节点的下一个节点指向自己（head.next.next = head）
  3. 断开当前节点的 next 指向，避免成环
  4. 返回新的头节点
