LeetCode #1 - 两数之和

Easy 哈希表 · 数组 · 经典入门

📋 题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 的那 两个 整数，并返回它们的数组下标。

注意：你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

示例 1：

输入：nums = [2, 7, 11, 15], target = 9
输出：[0, 1]
解释：因为 nums[0] + nums[1] == 9，返回 [0, 1]。

示例 2：

输入：nums = [3, 2, 4], target = 6
输出：[1, 2]

示例 3：

输入：nums = [3, 3], target = 6
输出：[0, 1]

💡 解题思路

这道题是 LeetCode 的第一题，也是算法面试中的「必考题」。核心思想是利用 哈希表 将查找时间从 O(n) 降到 O(1)。

暴力解法（O(n²)）

双重循环枚举所有可能的两个数组合，检查它们的和是否等于 target。时间复杂度高，不推荐。

哈希表解法（O(n)）⭐ 推荐

遍历数组时，对于当前数字 nums[i]，我们需要找 target - nums[i]。如果之前已经遍历过这个数字，就能直接返回答案。

用哈希表存储「数字 → 下标」的映射，一边遍历一边查询，实现 一次遍历 解决问题。

3        # 创建哈希表：数字 -> 下标


4        hash_map = {}


5        


6        # 遍历数组


7        for i, num in enumerate(nums):


8            # 计算需要找的数字


9            complement = target - num


10            


11            # 如果 complement 已在哈希表中，找到答案


12            if complement in hash_map:


13                return [hash_map[complement], i]


14            


15            # 将当前数字存入哈希表


16            hash_map[num] = i


17        


18        return []  # 无解（根据题意不会执行到这里）

📊 复杂度分析

暴力枚举 O(n²) O(1) 双重循环，不实用
哈希表（两遍扫描） O(n) O(n) 先存表，再查找
哈希表（一遍扫描）⭐ O(n) O(n) 边存边查，最优

🎯 关键点总结

  • 空间换时间：用哈希表 O(n) 空间换取 O(1) 查找时间
  • 一遍扫描：在存入当前元素之前先查表，实现单次遍历
  • complement 思想：对于 a + b = target，已知 a 则 b = target - a
  • 字典用法：Python 的 dict 就是哈希表实现，in 操作平均 O(1)
