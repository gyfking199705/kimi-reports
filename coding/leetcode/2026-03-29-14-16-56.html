LeetCode #56 - 合并区间

Merge Intervals | 中等难度 | 贪心算法 + 排序

📋 题目描述

以数组 intervals 表示若干个区间的集合，其中单个区间为 intervals[i] = [starti, endi]。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。


示例 1：

输入：intervals = [[1,3],[2,6],[8,10],[15,18]]
输出：[[1,6],[8,10],[15,18]]
解释：区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].

示例 2：

输入：intervals = [[1,4],[4,5]]
输出：[[1,5]]
解释：区间 [1,4] 和 [4,5] 可被视为重叠区间。

数组 排序 贪心

🎬 算法执行过程（点击步骤或代码行查看对应关系）

以输入 [[1,3],[2,6],[8,10],[15,18]] 为例，观察合并过程

Python 实现

class Solution:
    def merge(self, intervals: List[List[int]]) -> List[List[int]]:
        # 按区间起点升序排序
        intervals.sort(key=lambda x: x[0])
        
        merged = []
        for interval in intervals:
            # 如果当前区间与上一个不重叠，直接添加
            if not merged or merged[-1][1] < interval[0]:
                merged.append(interval)
            else:
                # 有重叠，合并区间，更新结束点
                merged[-1][1] = max(merged[-1][1], interval[1])
        
        return merged

💡 核心解题思路

  1. 排序先行：按区间起点升序排序，保证后续只需线性扫描
  2. 维护当前区间：用 merged[-1] 表示当前正在构建的合并后区间
  3. 判断重叠：当前区间起点 > 上个区间终点？不重叠 : 重叠
  4. 合并更新：重叠时更新终点为两者的最大值

关键洞察：排序后，任何可能与当前区间重叠的区间，其起点一定在当前区间起点之后（或相等）。因此只需比较上一个合并区间的终点与当前区间的起点。
