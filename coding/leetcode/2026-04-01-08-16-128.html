LeetCode #128 - 最长连续序列Medium

Longest Consecutive Sequence | 哈希表优化的经典之作

📋 题目描述

给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

示例 1:
输入: nums = [100, 4, 200, 1, 3, 2]
输出: 4
解释: 最长连续序列是 [1, 2, 3, 4]，长度为 4
示例 2:
输入: nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
输出: 9

要求：设计并实现时间复杂度为 O(n) 的算法。

🔍 寻找序列起点

💡 核心思路

关键洞察：不是每个数字都要去向后查找，只有当它是「序列起点」时才扩展。

如何判断序列起点？num - 1 不在集合中，说明没有更小的相邻数，这就是起点。

这样每个元素最多被访问两次（一次作为起点判断，一次在序列扩展中），保证 O(n) 复杂度。

🎬 算法执行过程（点击步骤或代码行查看对应关系）

Python 实现



1class Solution:


2    def longestConsecutive(self, nums: list[int]) -> int:


3        # 去重 + O(1) 查询


4        num_set = set(nums)


5        longest = 0


6        


7        for num in num_set:


8            # 只从序列起点开始


9            if num - 1 not in num_set:


10                current_num = num


11                current_streak = 1


12                


13                # 向后扩展序列


14                while current_num + 1 in num_set:


15                    current_num += 1


16                    current_streak += 1


17                


18                longest = max(longest, current_streak)


19        


20        return longest

📊 复杂度分析

复杂度类型 值 说明
时间复杂度 O(n) 每个数字最多入/出集合一次，序列扩展总次数有限
空间复杂度 O(n) 哈希集合存储所有不重复数字

为何是 O(n) 不是 O(n²)？

虽然外层是遍历，内层有 while 循环，但 num - 1 not in set 的条件保证了：只有序列起点才会触发 while。每个连续序列内的数字只会被 while 访问一次。总的时间 = 遍历 O(n) + 序列扩展 O(n) = O(n)。

🎯 面试要点

  • 这是「时间换空间」的反面：用 O(n) 空间换取 O(n) 时间
  • 排序解法是 O(n log n)，不满足题目要求
  • 关键在于「序列起点」的判断，避免重复计算
  • 问：如果要求返回序列本身而不是长度？答：记录起点即可
