LeetCode #15 - 三数之和

Medium 双指针 排序 去重

📋 题目描述

给你一个整数数组 nums，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k，同时还满足 nums[i] + nums[j] + nums[k] == 0。

请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

💡 核心考点：排序后利用双指针将O(n³)暴力解法优化到O(n²)，同时巧妙处理重复元素去重。

步骤1：排序数组

将数组排序：[-4, -1, -1, 0, 1, 2]

步骤2：固定第一个数

外层循环遍历，跳过重复元素

步骤3：去重判断

i>0 且 nums[i]==nums[i-1]时跳过

步骤4：初始化双指针

left = i+1, right = n-1

步骤5：双指针搜索

while left < right 循环

步骤6：计算三数之和

sum = nums[i] + nums[left] + nums[right]

步骤7：找到解

sum==0，加入结果，移动指针并去重

步骤8：和太小

sum < 0，left++ 增大和

步骤9：和太大

sum > 0，right-- 减小和

Python 实现



1class Solution:


2    def threeSum(self, nums: List[int]) -> List[List[int]]:


3        res, n = [], len(nums)


4        nums.sort()  # 排序，O(nlogn)


5        for i in range(n - 2):  # 固定第一个数


6            if i > 0 and nums[i] == nums[i - 1]:


7                continue  # 跳过重复的第一个数


8            if nums[i] > 0: break  # 剪枝：第一个数>0则不可能


9            left, right = i + 1, n - 1  # 双指针


10            while left < right:


11                s = nums[i] + nums[left] + nums[right]


12                if s == 0:


13                    res.append([nums[i], nums[left], nums[right]])


14                    while left < right and nums[left] == nums[left + 1]:


15                        left += 1  # 跳过重复的第二个数


16                    left += 1; right -= 1


17                elif s < 0:


18                    left += 1  # 和太小，左指针右移


19                else:


20                    right -= 1  # 和太大，右指针左移


21        return res

🎯 算法思路详解

核心思想：排序 + 双指针

这道题最直观的想法是三重循环暴力枚举，时间复杂度O(n³)。但通过先排序，可以将问题转化为"固定一个数，用双指针找另外两个数"，复杂度降为O(n²)。

关键技巧

  • 排序：排序后可以利用单调性使用双指针，同时方便去重
  • 三重去重：分别处理第一个数、左指针、右指针的重复情况
  • 剪枝：当nums[i] > 0时直接break，因为后面的数都更大
  • 双指针移动策略：sum < 0则left++增大，sum > 0则right--减小

时间复杂度分析

步骤复杂度说明
排序O(n log n)Python Timsort
外层循环O(n)固定第一个数
内层双指针O(n)每个元素最多被访问一次
总计O(n²)优于O(n³)的暴力解法

💭 面试高频追问

  • Q: 为什么要排序？不排序能做吗？
    A: 不排序可以使用哈希表，但去重更复杂。排序后利用单调性双指针更优雅。
  • Q: 空间复杂度是多少？
    A: 除了输出数组，只使用O(1)额外空间。如果考虑排序的递归栈，是O(log n)。
  • Q: 如果数组很大（10^6级别）会怎样？
    A: O(n²)会超时。此时需要考虑更高级的算法或数据结构设计。
