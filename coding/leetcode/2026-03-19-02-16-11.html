🎯 LeetCode #11 - 盛最多水的容器

Container With Most Water | 双指针 | 贪心算法经典入门

Medium 双指针 贪心 数组

📋 题目描述

给定一个长度为 n 的整数数组 height 。有 n 条垂线，第 i 条线的两个端点是 (i, 0) 和 (i, height[i]) 。

找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

示例 1

输入: height = [1,8,6,2,5,4,8,3,7]

输出: 49

解释: 选择第1条线(高度8)和第8条线(高度7)，宽度为7，高度为min(8,7)=7，面积=7×7=49

示例 2

输入: height = [1,1]

输出: 1

height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
← 选这两根线，面积最大 = 49

💡 核心洞察

水的容量 = min(height[left], height[right]) × (right - left)

由两根线中较矮的那根决定高度，两线之间的距离决定宽度。

关键观察：移动较高的线不可能得到更大的面积（宽度减小，高度受限于较矮的线）

解法 时间复杂度 空间复杂度 说明
暴力枚举 O(n²) O(1) 枚举所有两两组合
双指针（最优） O(n) O(1) 左右指针向中间移动

🧠 为什么双指针是正确的？

贪心策略证明

假设当前左指针在 i，右指针在 j，且 height[i] < height[j]。

为什么移动右指针不会漏掉最优解？

  • 当前面积 = height[i] × (j - i)
  • 如果固定 i，任何 k < j 的位置，面积 = min(height[i], height[k]) × (k - i)
  • 宽度 (k - i) < (j - i)，且高度 ≤ height[i]
  • 所以所有以 i 为左边界、k 为右边界的面积，都不可能超过当前面积
  • 因此可以安全地放弃 i，移动左指针

这就是贪心的精髓：每一步都做出局部最优选择，最终得到全局最优。

# 双指针解法 - 时间O(n) 空间O(1)
class Solution:
    def maxArea(self, height: List[int]) -> int:
        # 初始化双指针
        left, right = 0, len(height) - 1
        max_area = 0
        
        # 当左右指针未相遇时继续
        while left < right:
            # 计算当前面积：高度取较小值，宽度为下标差
            width = right - left
            h = min(height[left], height[right])
            current_area = width * h
            
            # 更新最大面积
            max_area = max(max_area, current_area)
            
            # 贪心：移动高度较小的指针
            if height[left] < height[right]:
                left += 1
            else:
                right -= 1
        
        return max_area
