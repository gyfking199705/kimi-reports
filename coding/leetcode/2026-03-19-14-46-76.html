LeetCode #76 - 最小覆盖子串

Hard 滑动窗口 哈希表

📋 题目描述

给定两个字符串 s 和 t，返回 s 中涵盖 t 所有字符的最小子串。

如果不存在符合条件的子串，返回空字符串 ""。

注意：对于 t 中重复字符，我们寻找的子串中该字符数量必须不少于 t 中该字符数量。

示例

输入：s = "ADOBECODEBANC", t = "ABC"

输出："BANC"

解释：子串 "BANC" 包含 'A'、'B'、'C' 各至少一次，且是最短的。

💡 核心思路：滑动窗口的"扩张-收缩"策略

先用右指针扩张窗口直到包含所有目标字符，再用左指针收缩窗口找最小值。两个哈希表分别记录需求（t）和窗口内现状（s窗口）。

🎬 算法执行过程（点击步骤查看代码高亮）

ADOBECODEBANC
1️⃣ 初始化
构建 need 哈希表，统计 t="ABC" 各字符需求：A:1, B:1, C:1
need = {A:1, B:1, C:1}
2️⃣ 窗口扩张（右指针移动）
right 从0开始遍历，加入 'A'，window[A]=1，满足 need[A] 要求
window[A:1], valid=1/3, 区间[0,0]
3️⃣ 继续扩张
right 移动到 'B'(索引3)，window[B]=1，valid=2/3
window[A:1,B:1], valid=2/3, 区间[0,3]
4️⃣ 首次满足条件
right 到 'C'(索引5)，valid=3/3，窗口 "ADOBEC" 包含所有字符
找到候选："ADOBEC" 长度6，区间[0,5]
5️⃣ 窗口收缩（左指针移动）
left=0 指向 'A'，移除后 window[A]=0 < need[A]，valid=2/3，停止收缩
收缩后区间[1,5]，长度5，记录最小值
6️⃣ 继续遍历
right 继续右移，经过 CODEB...直到索引10的 'A'，再次满足条件
区间[6,10] "CODEBA" 长度5
7️⃣ 找到最优解
继续收缩，left 到索引9的 'B'，窗口 "BANC" 长度4，更新最小值
最优解："BANC" 长度4，区间[9,12]
8️⃣ 返回结果
遍历结束，返回最短子串 "BANC"
结果："BANC"

Python 实现



1from collections import defaultdict


2


3class Solution:


4    def minWindow(self, s: str, t: str) -> str:


5        need = defaultdict(int)      # t 中字符需求


6        for c in t: need[c] += 1


7        window = defaultdict(int)    # 当前窗口字符统计


8        left = valid = 0             # valid: 满足条件的字符数


9        start, min_len = 0, float('inf')


10


11        for right, c in enumerate(s):  # 右指针扩张


12            if c in need:


13                window[c] += 1


14                if window[c] == need[c]:


15                    valid += 1


16


17            while valid == len(need):   # 左指针收缩


18                if right - left + 1 < min_len:


19                    start, min_len = left, right - left + 1


20


21                d = s[left]              # 要移除的字符


22                left += 1


23                if d in need:


24                    if window[d] == need[d]:


25                        valid -= 1


26                    window[d] -= 1


27


28        return "" if min_len == float('inf') else s[start:start + min_len]

初始化 need/window 右指针扩张 valid == len(need)? 左指针收缩 更新最小值 返回结果

📊 复杂度分析

指标复杂度说明
时间复杂度 O(|S| + |T|) 左右指针各遍历 s 一次，每个字符最多被访问两次
空间复杂度 O(|Σ|) 哈希表存储字符，Σ 为字符集大小（ASCII约128）

🔑 关键技巧总结

1. 双哈希表设计

need 记录目标需求，window 记录当前状态。分离关注点，逻辑清晰。

2. valid 计数器

不用每次比较整个哈希表，用 valid 记录"已满足需求的字符种类数"，O(1) 判断窗口有效性。

3. 收缩时机

一旦 valid == len(need) 立即收缩左边界，寻找以当前 right 结尾的最小窗口。
