🚀 LeetCode #55 - 跳跃游戏

贪心算法 | 数组 | Medium | 可达性问题的经典模板

📋 题目描述

给你一个非负整数数组 nums ，你最初位于数组的 第一个下标 0 处。

数组中的每个元素代表你在该位置可以跳跃的 最大长度。

判断你是否能够到达最后一个下标。

🎯 核心问题：不是求最少步数，也不是求具体路径，而是判断「能否到达终点」。

示例 1：

输入: nums = [2,3,1,1,4]
输出: true
解释: 可以先跳 1 步，从下标 0 到达下标 1，然后再从下标 1 跳 3 步到达最后一个下标。

示例 2：

输入: nums = [3,2,1,0,4]
输出: false
解释: 无论如何，总会到达下标为 3 的位置。但该下标处的最大跳跃长度是 0，所以永远不可能到达最后一个下标。

约束条件：

  • 1 <= nums.length <= 10^4
  • 0 <= nums[i] <= 10^5

💡 核心思路：贪心思想——维护最远可达距离

这道题的关键在于：不需要知道具体怎么跳，只需要知道「最远能到哪里」。

贪心洞察：遍历数组时，维护一个变量 max_reach 表示从起点出发能够到达的最远位置。如果在遍历过程中发现当前位置已经超过了 max_reach，说明这个位置是无法到达的，直接返回 false。

为什么这是正确的？ 因为只要当前位置 i <= max_reach，就说明从起点可以通过某条路径到达位置 i。而从位置 i，我们可以跳到 i + nums[i]，所以更新 max_reach = max(max_reach, i + nums[i])。

📝 关键易错点

  • 循环条件：应该是 range(len(nums) - 1) 还是 range(len(nums))？
    • 答案是都可以，但如果已经能到达最后一个位置，可以提前返回
    • 常见优化：for i in range(len(nums)): 内判断 if i > max_reach: return False
  • 更新时机：max_reach = max(max_reach, i + nums[i]) 要在检查可达性之后还是之前？
    • 先检查 i > max_reach，再更新 max_reach
  • 边界情况：
    • 数组长度为1时，起点就是终点，直接返回 True
    • 第一个元素为0且数组长度>1时，无法移动，返回 False
