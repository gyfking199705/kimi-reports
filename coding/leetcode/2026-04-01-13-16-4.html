寻找两个正序数组的中位数

📋 题目描述

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log(m+n))。

💡 核心考点：二分查找的经典变体。将"找中位数"转化为"找第k小的数"，在较小的数组中进行二分，每次排除一半不可能的元素。

示例 1：

输入：nums1 = [1,3], nums2 = [2]
输出：2.00000
解释：合并数组 = [1,2,3] ，中位数 2

示例 2：

输入：nums1 = [1,2], nums2 = [3,4]
输出：2.50000
解释：合并数组 = [1,2,3,4] ，中位数 (2 + 3) / 2 = 2.5

Python 实现

🎯 核心思想：将找中位数转化为"找第k小的数"。在较小的数组A中取前k/2个元素，在数组B中也取前k/2个元素，比较两个部分的最后一个元素，较小的那部分不可能包含第k小的数，可以排除。

📊 复杂度分析

方法 时间复杂度 空间复杂度 说明
合并数组法 O(m+n) O(m+n) 不符合题目要求，但简单直观
双指针法 O(m+n) O(1) 只需遍历到中间位置，但仍是线性
二分查找法 O(log(m+n)) O(1) ✅ 最优解法

💡 解题技巧总结

  • 转化思维：中位数 = 合并后第 (m+n)/2 或 (m+n)/2+1 小的数
  • 二分技巧：永远选择较短的数组进行二分，提高效率
  • 边界处理：注意数组为空、越界等特殊情况
  • k的处理：奇偶长度统一处理，递归/迭代找第k小的数
