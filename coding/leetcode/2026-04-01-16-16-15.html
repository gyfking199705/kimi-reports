LeetCode #15 - 三数之和

Medium 数组 双指针 排序

📋 题目描述

给你一个整数数组 nums ，判断是否存在三元组 [nums[i], nums[j], nums[k]] 满足 i != j、i != k 且 j != k ，同时还满足 nums[i] + nums[j] + nums[k] == 0。

请你返回所有和为 0 且不重复的三元组。

注意：答案中不可以包含重复的三元组。

示例：
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2], [-1,0,1]]

解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 ✓
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 ✓ (重复，忽略)
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 ✓

💡 核心思路

关键洞察：排序后，固定一个数，剩下两数用双指针从两端向中间逼近。这样可以将 O(n³) 暴力解法优化到 O(n²)。
  • 排序：排序是前提，便于使用双指针和去重
  • 固定 + 双指针：固定 nums[i]，在 (i+1, n-1) 范围内找两数之和等于 -nums[i]
  • 去重：跳过重复元素，避免结果中出现重复三元组
  • 剪枝：利用排序后的单调性提前终止不可能的情况

📊 复杂度分析

维度 复杂度 说明
时间复杂度 O(n²) 排序 O(n log n) + 双指针遍历 O(n²)，总体 O(n²)
空间复杂度 O(1) / O(log n) 忽略结果数组，仅使用常数额外空间（排序栈空间 O(log n)）
面试追问：
• 如果数组已排序，时间复杂度是多少？→ 仍是 O(n²)，排序不是瓶颈
• 如果要求返回的是下标而不是数值？→ 排序会破坏原下标，需要用哈希表做法（更复杂）
• 扩展：四数之和怎么做？→ 固定两个数，剩下两个用双指针，O(n³)

📝 关键点总结

  • 排序是双指针的前提 —— 没有排序，双指针无从下手
  • 去重是难点 —— 三个位置都要去重（固定位置、左指针、右指针）
  • 剪枝优化 —— nums[i] > 0 时直接 break，利用单调性
  • 空间换时间 vs 时间换空间 —— 本题用双指针 O(1) 空间，哈希表做法需要 O(n) 空间
