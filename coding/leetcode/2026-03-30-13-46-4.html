LeetCode #4 - 寻找两个正序数组的中位数

Hard 二分查找 数组 分治算法

📋 题目描述

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log (m+n)) 。

💡 核心洞察

中位数本质上是「中间的数」，意味着需要将所有数分成左右两半，左半部分的最大值 ≤ 右半部分的最小值。对于两个有序数组，关键是找到一个分割点使得两边元素数量平衡。

🎬 算法执行过程（点击步骤查看代码联动）

执行步骤

1. 初始化与输入检查

确保 nums1 是较短的数组，在较短的数组上进行二分，保证时间复杂度 O(log(min(m,n)))

nums1: [1,3], nums2: [2]

2. 二分查找设置

初始化二分查找的左右边界，准备在 nums1 上找分割点

left=0, right=2 (nums1长度)

3. 计算分割点

i = nums1的分割位置，j = nums2的分割位置，保证左右两部分元素总数相等

i=1, j=1, partition1=3, partition2=2

4. 检查分割有效性

左半部分的最大值 ≤ 右半部分的最小值 时，分割正确

maxLeft(3) ≤ minRight(2)? 否，需要调整

5. 调整搜索范围

nums1的左半部分太大，需要减小i，将右边界左移

right = i - 1 = 0

6. 重新计算分割点

新的分割位置，i=0 表示 nums1 所有元素都在右侧

i=0, j=2, maxLeft1=-∞, minRight1=1

7. 找到正确分割

满足条件：maxLeft1 ≤ minRight2 且 maxLeft2 ≤ minRight1

-∞ ≤ 2 ✓ 且 2 ≤ 1? 不，继续检查边界情况

8. 计算中位数

总长度为奇数，中位数是右半部分的最小值

result = 2.0

Python 实现

1class Solution:


2    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:


3        # 确保nums1是较短的数组


4        if len(nums1) > len(nums2):


5            nums1, nums2 = nums2, nums1


6        


7        m, n = len(nums1), len(nums2)


8        total_left = (m + n + 1) // 2  # 左半部分需要的元素个数


9        


10        left, right = 0, m  # 在nums1上二分查找分割点


11        


12        while left <= right:


13            i = (left + right) // 2  # nums1的分割位置


14            j = total_left - i       # nums2的分割位置


15            


16            # 处理边界情况


17            maxLeft1 = float('-inf') if i == 0 else nums1[i - 1]


18            minRight1 = float('inf') if i == m else nums1[i]


19            maxLeft2 = float('-inf') if j == 0 else nums2[j - 1]


20            minRight2 = float('inf') if j == n else nums2[j]


21            


22            # 检查是否找到正确分割


23            if maxLeft1 <= minRight2 and maxLeft2 <= minRight1:


24                # 找到正确分割，计算中位数


25                if (m + n) % 2 == 0:


26                    return (max(maxLeft1, maxLeft2) + min(minRight1, minRight2)) / 2


27                else:


28                    return max(maxLeft1, maxLeft2)


29            


30            elif maxLeft1 > minRight2:


31                # nums1左半部分太大，需要左移


32                right = i - 1


33            else:


34                # nums1左半部分太小，需要右移


35                left = i + 1


36        


37        return 0.0  # 不会执行到这里

📊 复杂度分析

方法 时间复杂度 空间复杂度 说明
暴力合并 O(m+n) O(m+n) 合并后排序再找中位数，不符合要求
双指针找第k小 O(m+n) O(1) 模拟合并过程，找到中位数位置停止
二分查找（本题最优） O(log(min(m,n))) O(1) 在较短数组上二分，满足题目要求

🎯 关键理解要点

  • 为什么要在较短的数组上二分？ 保证时间复杂度为 O(log(min(m,n)))，同时避免分割点超出范围。
  • 分割点的含义： i 表示 nums1 中有 i 个元素在左半部分，j = total_left - i 是 nums2 的分割点。
  • 边界情况处理： 当 i=0 或 i=m 时，需要用 -inf 或 inf 作为边界值。
  • 正确分割的条件： maxLeft1 ≤ minRight2 且 maxLeft2 ≤ minRight1，确保左半部分所有元素 ≤ 右半部分所有元素。

🔍 核心思想可视化

将两个数组分别在某位置分割，左半部分包含 (m+n+1)//2 个元素。中位数只取决于四个边界值：

nums1: [ ... maxLeft1 | minRight1 ... ]
nums2: [ ... maxLeft2 | minRight2 ... ]

左半部分最大值 = max(maxLeft1, maxLeft2)
右半部分最小值 = min(minRight1, minRight2)
