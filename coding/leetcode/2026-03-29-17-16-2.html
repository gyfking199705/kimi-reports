LeetCode #2 - 两数相加

Medium 链表 数学 模拟

📋 题目描述

给你两个 非空 的链表，表示两个非负的整数。它们每位数字都是按照 逆序 的方式存储的，并且每个节点只能存储 一位 数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

示例 1
输入：l1 = [2,4,3], l2 = [5,6,4]
输出：[7,0,8]
解释：342 + 465 = 807
对应链表表示：7 → 0 → 8
示例 2
输入：l1 = [0], l2 = [0]
输出：[0]
示例 3
输入：l1 = [9,9,9,9,9,9,9], l2 = [9,9,9,9]
输出：[8,9,9,9,0,0,0,1]
解释：9999999 + 9999 = 10009998

💡 核心洞察

为什么逆序存储反而简单？

因为加法是从个位（最低位）开始的！链表的头节点正好是个位，与我们做加法的顺序完全一致。这意味着我们可以同步遍历两个链表，就像小学竖式加法一样，从右往左逐位计算。

关键设计决策

  • 进位处理：用一个变量 carry 记录进位（0 或 1）
  • 循环条件：只要 l1、l2 任一不为空，或进位不为 0，就继续
  • 链表长度不等：缺失的位视为 0
  • 最高位进位：循环结束后如果 carry > 0，需要新增一个节点

📊 复杂度分析

维度 值 说明
时间复杂度 O(max(m, n)) 遍历两个链表较长的那个的长度，m 和 n 分别是 l1 和 l2 的长度
空间复杂度 O(max(m, n)) 结果链表的长度最多为 max(m, n) + 1（考虑最高位进位）
空间优化 O(1) 如果允许修改输入链表，可以复用 l1 或 l2 的节点（面试进阶）

🎯 面试要点

  • 边界条件：如何处理链表长度不同的情况？（答案：缺失位补0）
  • 最后的进位：循环结束后 carry 不为0怎么办？（答案：新建节点）
  • 空间优化：能否在 O(1) 空间完成？（答案：可以，复用输入链表节点）
  • 变式思考：如果是正序存储的数字呢？（答案：用栈或先翻转链表）
  • 代码规范：哑节点（dummy node）的使用可以简化边界处理
