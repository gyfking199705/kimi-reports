LeetCode #55 - 跳跃游戏

Medium 贪心算法 · 数组 · 动态规划

📋 题目描述

给定一个非负整数数组 nums ，你最初位于数组的 第一个下标 。

数组中的每个元素代表你在该位置可以跳跃的 最大长度 。

判断你是否能够到达最后一个下标。

核心洞察：不需要模拟每一步的跳跃，只需关心「最远能到达哪里」。如果当前位置在可达范围内，就更新最远可达距离。

示例

输入: nums = [2,3,1,1,4]
输出: true
解释: 可以先跳 1 步，从下标 0 到达下标 1，然后再从下标 1 跳 3 步到达最后一个下标。

输入: nums = [3,2,1,0,4]
输出: false
解释: 无论怎样，你总会到达下标为 3 的位置。但该下标的最大跳跃长度是 0，所以你永远不可能到达最后一个下标。

📊 复杂度分析

算法 时间复杂度 空间复杂度 说明
贪心（推荐） O(n) O(1) 只需遍历一次，维护最远可达位置
动态规划 O(n²) O(n) dp[i]表示能否到达i，对每个位置检查前面所有可达位置
回溯（超时） O(2ⁿ) O(n) 枚举所有跳跃组合，不推荐

💡 关键思路

贪心策略：维护一个变量 max_reach 表示当前能够到达的最远位置。

  • 遍历数组的每个位置 i
  • 如果 i > max_reach，说明当前位置不可达，直接返回 false
  • 否则，更新 max_reach = max(max_reach, i + nums[i])
  • 如果 max_reach >= n-1，提前返回 true
为什么贪心是对的？
我们不关心具体怎么跳到位置 i，只关心「能不能跳到」。只要 i 在可达范围内（i ≤ max_reach），那么从 i 出发能到达的所有位置也都是可达的。这是一个典型的贪心选择性质——局部最优（尽可能延伸可达范围）导致全局最优（到达终点）。
