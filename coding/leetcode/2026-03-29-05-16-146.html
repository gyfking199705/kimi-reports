LeetCode #146 - LRU 缓存

LRU Cache · 经典设计题 · 哈希表 + 双向链表

Medium 🎨 设计 🔗 哈希表 🔄 双向链表

📋 题目描述

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。

实现 LRUCache 类：

  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value；如果不存在，则向缓存中插入该组 key-value。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字

函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

🧠 核心思路

为什么用 哈希表 + 双向链表？

• 哈希表 → O(1) 时间查找 key 是否存在

• 双向链表 → O(1) 时间移动/删除节点（记录使用顺序）

• 链表头部 = 最近使用，链表尾部 = 最久未使用

Python 实现 时间 O(1) · 空间 O(capacity)

📊 复杂度分析

操作 时间复杂度 空间复杂度 说明
get(key) O(1) O(1) 哈希表查找 + 链表移动头部
put(key, value) O(1) O(1) 哈希表插入/更新 + 链表操作
整体 O(1) O(capacity) 最多存储 capacity 个键值对

💡 关键要点

  • 使用 伪头部 和 伪尾部 节点简化边界处理，避免空指针判断
  • 每次访问（get/put已存在）都要将节点移动到链表头部，标记为最近使用
  • 容量满时删除尾部节点，同时从哈希表中移除对应的 key
  • 双向链表使得删除和插入操作都可以在 O(1) 时间完成
  • Python 可用 collections.OrderedDict 一行实现，但面试建议手写数据结构
