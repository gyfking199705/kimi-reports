LeetCode #76 - 最小覆盖子串

Hard 滑动窗口 哈希表 双指针

📋 题目描述

给你一个字符串 s 、一个字符串 t 。返回 s 中涵盖 t 所有字符的最小子串。如果不存在涵盖 t 所有字符的子串，则返回空字符串 ""。

注意：对于 t 中重复字符，我们寻找的子字符串中该字符数量必须不少于 t 中该字符数量。
如果 s 中存在这样的子串，我们保证它是唯一的答案。
示例 1：
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
示例 2：
输入：s = "a", t = "a"
输出："a"
示例 3：
输入：s = "a", t = "aa"
输出：""
解释：t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子字符串，返回空字符串。

🧠 核心思路：滑动窗口

🔑 算法精髓

滑动窗口是解决子串问题的利器。核心思想：

  1. 扩张窗口：右指针右移，将字符纳入窗口，直到窗口包含 t 中所有字符
  2. 收缩窗口：左指针右移，尝试缩小窗口，同时保持覆盖条件
  3. 记录最优：在收缩过程中记录满足条件的最小窗口
关键问题：如何判断窗口已覆盖 t 中所有字符？
使用哈希表记录 t 中各字符需求量，再用一个计数器 need 记录还需满足的字符种类数。当 need == 0 时，窗口有效。

Python 实现 (点击行号查看对应步骤)


1class Solution:
2    def minWindow(self, s: str, t: str) -> str:
3        # 统计t中各字符需求量
4        need_map = {}
5        for c in t:
6            need_map[c] = need_map.get(c, 0) + 1
7        need = len(need_map)  # 还需满足的字符种类数
8        
9        left = 0  # 窗口左边界
10        min_len = float('inf')
11        min_start = 0
12        window_map = {}  # 窗口内字符计数
13        
14        # 右指针扩张窗口
15        for right in range(len(s)):
16            char = s[right]
17            window_map[char] = window_map.get(char, 0) + 1
18            
19            # 当前字符满足需求量
20            if char in need_map and window_map[char] == need_map[char]:
21                need -= 1
22            
23            # 窗口已覆盖所有字符，尝试收缩
24            while need == 0:
25                # 更新最小窗口
26                if right - left + 1 < min_len:
27                    min_len = right - left + 1
28                    min_start = left
29                
30                # 左指针右移，收缩窗口
31                left_char = s[left]
32                window_map[left_char] -= 1
33                if left_char in need_map and window_map[left_char] < need_map[left_char]:
34                    need += 1
35                left += 1
36        
37        return s[min_start:min_start + min_len] if min_len != float('inf') else ""

⏱️ 复杂度分析

O(n)
时间复杂度
左右指针各遍历字符串一次，每个字符最多被访问两次
O(k)
空间复杂度
k为字符集大小，使用两个哈希表存储字符计数
为什么时间是O(n)？
虽然代码中有嵌套循环（for + while），但左指针 left 和右指针 right 都是从0开始只向右移动，每个字符最多被访问两次（被right经过一次，被left经过一次），所以总体是线性时间。

💡 易错点与面试提示

坑点 正确做法
混淆字符种类数和字符总数 need 记录的是字符种类数，不是总数。如t="AAB"，种类数是2(A和B)，不是3
收缩时误判条件 收缩后只有当窗口内字符数小于需求量时才增加need，等于时说明仍然满足
没有处理无解情况 最后判断min_len是否仍为inf，是则返回空串
窗口更新时机 在收缩前更新最小窗口，因为当前窗口是满足条件的

📝 总结

最小覆盖子串是滑动窗口算法的经典代表，核心在于：

  1. 使用双指针维护动态窗口，右指针扩张，左指针收缩
  2. 哈希表记录字符需求量，need计数器追踪满足状态
  3. 扩张时更新状态，满足条件时收缩并记录最优解
