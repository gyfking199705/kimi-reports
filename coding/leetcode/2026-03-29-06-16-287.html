LeetCode #287 - 寻找重复数

Medium 二分查找 Floyd判圈算法 链表

📋 题目描述

给定一个包含 n + 1 个整数的数组 nums，其数字都在 [1, n] 范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数，返回这个重复的数。

你设计的解决方案必须不修改数组 nums 且只用常量级 O(1) 的额外空间。

约束条件：
  • 1 ≤ n ≤ 105
  • nums.length == n + 1
  • 1 ≤ nums[i] ≤ n
  • nums 中只有一个重复的数字，但它可能重复多次

💡 核心思路：将数组视为链表

关键洞察：数组可以看作一个隐式链表

由于数组中的值都在 [1, n] 范围内，而数组长度为 n+1，我们可以将数组看作一个函数图：

  • 索引 i 指向 nums[i]
  • 每个索引指向一个值，这个值又可以作为下一个索引

因为存在重复数字，必然有两个不同的索引指向同一个值，形成一个环。问题转化为：找到环的入口。

🎯 为什么 Floyd 判圈算法适用？

这是一个经典的 "Floyd's Tortoise and Hare" 应用场景：

  1. 阶段一：快慢指针相遇 - 快指针每次走2步，慢指针每次走1步，必在环内相遇
  2. 阶段二：找环入口 - 将慢指针移回起点，两指针同速前进，相遇点即环入口

数学证明：设环前有 a 步，环长为 b，相遇时慢指针走了 s 步，则快指针走了 2s 步。2s = s + kb（k为圈数），所以 s = kb。相遇点到环入口的距离 = (a + kb) - a = kb - a mod b。
