LeetCode #230 - 二叉搜索树中第K小的元素

Kth Smallest Element in a Binary Search Tree | 难度: Medium | 高频面试题

📋 题目描述

给定一个二叉搜索树的根节点 root，和一个整数 k，请你设计一个算法查找其中第 k 小的元素（从 1 开始计数）。

🔑 BST 性质：二叉搜索树的中序遍历（左→根→右）结果是有序的（升序）。因此第 k 小的元素就是中序遍历序列中的第 k 个元素。

示例 1：

输入：root = [3,1,4,null,2], k = 1
输出：1

示例 2：

输入：root = [5,3,6,2,4,null,null,1], k = 3
输出：3

约束条件：

  • 树中的节点数为 n
  • 1 ≤ k ≤ n ≤ 10⁴
  • 0 ≤ Node.val ≤ 10⁴

Python 实现 - 中序遍历迭代法



1# Definition for a binary tree node.


2# class TreeNode:


3#     def __init__(self, val=0, left=None, right=None):


4#         self.val = val


5#         self.left = left


6#         self.right = right


7class Solution:


8    def kthSmallest(self, root: TreeNode, k: int) -> int:


9        stack = []          # 模拟递归栈


10        curr = root         # 当前节点指针


11        count = 0           # 已访问节点计数


12        

13        while stack or curr:     # 栈不空或当前节点存在


14            # 1. 一直向左走，将路径节点压栈


15            while curr:


16                stack.append(curr)


17                curr = curr.left


18            

19            # 2. 弹出栈顶，处理当前节点


20            curr = stack.pop()


21            count += 1


22            

23            # 3. 检查是否找到第k小


24            if count == k:


25                return curr.val


26            

27            # 4. 转向右子树继续


28            curr = curr.right


29        

30        return -1  # 理论上不会执行到这里

递归法实现（更简洁）



31class Solution:


32    def kthSmallest(self, root: TreeNode, k: int) -> int:


33        self.count = 0


34        self.result = 0


35        self.inorder(root, k)


36        return self.result


37    

38    def inorder(self, node: TreeNode, k: int):


39        if not node or self.count >= k:


40            return


41        self.inorder(node.left, k)   # 遍历左子树


42        self.count += 1              # 访问当前节点


43        if self.count == k:          # 找到目标


44            self.result = node.val


45            return


46        self.inorder(node.right, k)  # 遍历右子树

📊 复杂度分析

解法时间复杂度空间复杂度说明
中序遍历 - 迭代O(H + k)O(H)H为树高，平均O(log n)，最坏O(n)
中序遍历 - 递归O(H + k)O(H)递归栈深度，同迭代
完全遍历（存储所有）O(n)O(n)不推荐，浪费空间

最优解：迭代/递归中序遍历，当访问到第 k 个节点时立即返回，无需遍历整棵树。

💡 关键思路

  • BST核心性质：中序遍历结果是有序的（升序排列）
  • 提前终止：不需要遍历完整棵树，访问到第 k 个节点即可返回
  • 迭代vs递归：迭代法空间利用更可控，递归法代码更简洁
  • 进阶优化：若需要频繁查询不同 k 值，可在节点中维护子树大小，实现 O(log n) 查询
