LeetCode #146 - LRU缓存机制

设计并实现一个满足 LRU (最近最少使用) 缓存约束的数据结构

📋 题目描述

Medium 哈希表 双向链表 设计

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。

实现 LRUCache 类：

  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value；如果不存在，则向缓存中插入该组 key-value。如果插入操作导致关键字数量超过 capacity，则应该 逐出 最久未使用的关键字。

函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。

💡 核心思路

为什么需要双向链表 + 哈希表？

LRU需要维护使用顺序，链表可以O(1)移动节点。但链表查找是O(n)，所以用哈希表记录key到节点的映射，实现O(1)查找。

数据结构选择：

  • 双向链表：维护使用顺序，最近使用的放在头部，最久未使用的在尾部
  • 哈希表 (dict)：存储 key → 链表节点的映射，实现O(1)查找

关键操作：

  • get：哈希表找到节点 → 移到链表头部 → 返回值
  • put：存在则更新值+移头部；不存在则新建 → 超容则删尾部

🎯 面试要点

  • 为什么用双向链表？ 单向链表删除节点需要找到前驱节点O(n)，双向链表O(1)
  • 为什么用哈希表？ 链表不支持O(1)按key查找
  • 边界情况：容量为0、重复put相同key、get不存在的key
  • Python简化：可以用 collections.OrderedDict 或 functools.lru_cache 实现
